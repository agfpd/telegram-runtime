# 02 — Быстрый старт

**Русский** · [English](../02-quickstart.md)

От установки до первого сообщения. Под каждым шагом — что он делает.

## Требования

- **macOS** и установленный, работающий [iapeer](https://github.com/agfpd/iapeer). Мост опирается на обмен сообщениями iapeer и реестр пиров — без iapeer он не работает.
- **Бот Telegram** — токен от `@BotFather` (создаётся бесплатно за минуту в самом Telegram).

## Установка пакета

```bash
npx @agfpd/telegram-runtime
```

Установка раскладывает бинарь `telegram-runtime` и пишет манифест рантайма, по которому iapeer знает о нём. То же делает `iapeer install-runtime telegram`. Установка идемпотентна.

## Присутствие человека

Чтобы человек присутствовал в Telegram, ему нужен пир-человек с прописанным Telegram-идентификатором. Обычно его создаёт онбординг iapeer (`iapeer onboard` заводит пира-владельца в Telegram). Отдельно человека заводят так:

```bash
iapeer create <имя-человека> --runtime telegram
```

В отличие от агентов, у telegram-рантайма нет заранее объявленного набора пиров — людей добавляет оператор по одному.

## Подключение бота к агенту

```bash
iapeer connect telegram <агент> --token <токен-бота>
```

Одна команда делает всё: регистрирует бота (проверяя токен у Telegram и забирая его настоящее имя), прописывает привязку бота к агенту в профиль, перезапускает мост, чтобы он подхватил нового бота. От вас нужен только токен.

## Первое сообщение

После подключения **отправьте боту первое сообщение из Telegram**. Это обязательно: по правилам Telegram бот не может написать человеку первым — чат должен начать человек. После первого входящего канал оживает, и дальше общение идёт в обе стороны.

## Проверка

```bash
telegram-runtime doctor
```

Команда проверяет цепочку: профиль на месте, Telegram-идентификатор задан, все боты настроены и привязаны, нет «осиротевших» ботов без агента. С флагом `--json` — машинно-читаемо.
