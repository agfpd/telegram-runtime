# telegram-runtime

**Русский** · [English](../README.md)

telegram-runtime — мост между Telegram и командой агентов [iapeer](https://github.com/agfpd/iapeer). Он даёт человеку присутствие в системе через Telegram: владелец пишет агенту из мессенджера и получает ответы туда же. Для агента человек — обычный пир, которому он шлёт `send_to_peer`; мост превращает это в сообщение в нужном Telegram-чате и обратно.

```text
   человек в Telegram                          агент (Claude/Codex)
        │  пишет боту                               │  send_to_peer(arthur, "…")
        ▼                                            ▼
   ┌──────────────────────────────────────────────────────────┐
   │  telegram-runtime — мост                                   │
   │  входящее: Telegram → пир-адресат                          │
   │  исходящее: пир → Telegram-чат                             │
   │  голос · печатает-индикатор · вложения · команды           │
   └──────────────────────────────────────────────────────────┘
```

Это `*-runtime` пакет экосистемы iapeer: рантайм присутствия для пиров-людей, наравне с агентскими рантаймами Claude и Codex. Агенты Telegram-инструментов не получают — для них канал прозрачен, они просто пишут пиру по имени.

## Что умеет

- **Двусторонний обмен** — входящие от человека доставляются пиру-адресату, ответы пира уходят в его Telegram-чат.
- **Голос** — голосовое от человека расшифровывается в текст и доезжает пиру; агент может ответить голосовым файлом.
- **Индикатор «печатает»** — пока агент работает над ответом, в чате виден индикатор набора.
- **Лента активности** — по желанию владелец видит, что агент делает прямо сейчас (какими инструментами пользуется).
- **Вложения** — файлы и картинки ходят в обе стороны.
- **Команды управления** — `/stop`, `/new`, `/compact` и алиасы-шорткаты прямо из чата.
- **Несколько ботов** — у каждого агента может быть свой бот, мост опрашивает их все одним процессом.

## Быстрый старт

Требуется установленный и работающий [iapeer](https://github.com/agfpd/iapeer).

```bash
npx @agfpd/telegram-runtime                       # установить пакет
iapeer connect telegram <агент> --token <токен>   # привязать бота к агенту
# затем — отправьте боту первое сообщение из Telegram (бот не может начать чат первым)
```

Подробно — в [02 — Быстрый старт](02-быстрый-старт.md).
