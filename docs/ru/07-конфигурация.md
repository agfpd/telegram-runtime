# 07 — Конфигурация

**Русский** · [English](../07-configuration.md)

telegram-runtime работает из коробки без настройки. Всё ниже опционально — переменные окружения, которые включают расшифровку голоса, направляют трафик через прокси или подкручивают поведение. Задаются в окружении запуска моста.

## Расшифровка голоса (STT)

Без STT-сервиса голосовое доходит до агента файлом. Чтобы расшифровывать в текст, укажите мосту адрес OpenAI-совместимого endpoint'а транскрипции (например локального `speaches` или `faster-whisper-server`):

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `TELEGRAM_STT_ENDPOINT` | — | базовый URL OpenAI-совместимого сервиса `/v1/audio/transcriptions`; пусто отключает основной путь |
| `TELEGRAM_STT_MODEL` | дефолт endpoint'а | имя модели, отправляемое сервису |
| `TELEGRAM_STT_LANGUAGE` | автоопределение | подсказка языка (ISO-639-1: `en`, `ru`, …) |
| `TELEGRAM_STT_PROMPT` | — | необязательная затравка декодеру |
| `TELEGRAM_STT_TIMEOUT_MS` | 30000 | таймаут запроса к endpoint'у |

Если endpoint недоступен или не задан, срабатывает локальный fallback:

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `TELEGRAM_STT_FALLBACK_CMD` | `mlx_whisper` | локальная команда транскрипции; пусто отключает |
| `TELEGRAM_STT_FALLBACK_MODEL` | дефолт команды | флаг модели для fallback-команды |
| `TELEGRAM_STT_KEEP_FILE` | выкл. | `1` прикладывает исходный аудио-файл рядом с расшифровкой |

Если и endpoint, и fallback не справились, голос доходит файлом. Расшифровка доставляется с пометкой, что это была речь.

## Индикаторы и оформление

Все три включены по умолчанию; чтобы отключить — задайте `0`:

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `TELEGRAM_TYPING` | вкл. | индикатор «печатает», пока агент работает |
| `TELEGRAM_ACTIVITY` | вкл. | лента активности, общий выключатель на хост (пер-пировый toggle — `/activity` в чате) |
| `TELEGRAM_ACTIVITY_DEFAULT` | вкл. | значение ленты по умолчанию для пира, который его не задал |
| `TELEGRAM_RICH` | вкл. | сообщения с богатым оформлением (markdown на стороне сервера) |

## Поверхности демона для владельца

Обе включены по умолчанию и гасятся раздельно намеренно: заглушить карточки подтверждений не должно означать заглушить сообщения о немоте — это канал последней надежды, ведь пир, который сообщил бы о себе, ровно тот, который не может говорить.

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `TELEGRAM_APPROVAL` | вкл. | карточки подтверждений ([04 — Возможности](04-возможности.md#подтверждение-человеком)) |
| `TELEGRAM_NOTICES` | вкл. | уведомления о немых пирах ([04 — Возможности](04-возможности.md#уведомления-о-немых-пирах)) |
| `TELEGRAM_APPROVAL_LOG` | вкл. | структурные события подтверждений и соединения в stderr |
| `TELEGRAM_NOTICE_LOG` | вкл. | структурные события уведомлений в stderr |

## Прокси

Если API Telegram требует прокси, мост берёт первую заданную из:

```
TELEGRAM_RUNTIME_PROXY  →  HTTPS_PROXY  →  HTTP_PROXY
```

## Тонкая настройка

Эти почти никогда не нужны — дефолты выверены. Перечислены, чтобы поведение не оставалось загадкой.

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `TELEGRAM_TYPING_POLL_MS` | 3000 | как часто опрашивается состояние занятости агента |
| `TELEGRAM_TYPING_MIN_MS` | 5000 | минимальное время показа индикатора набора |
| `TELEGRAM_TYPING_CAP_MS` | 1800000 | жёсткий потолок индикатора набора (30 мин) |
| `TELEGRAM_PANELOG_BUSY_MS` | 4000 | возраст лога активности, ниже которого агент считается занятым |
| `TELEGRAM_ACTIVITY_POLL_MS` | 500 | как часто читается транскрипт на новые шаги |
| `TELEGRAM_ACTIVITY_EDIT_MS` | 1000 | троттлинг правки сообщения ленты |
| `TELEGRAM_ACTIVITY_MAX_STEPS` | 30 | сколько последних шагов показывает лента |
| `TELEGRAM_OUTBOUND_TIMEOUT_MS` | 30000 | таймаут одной отправки в Telegram |
| `TELEGRAM_OUTBOUND_RETRIES` | 2 | повторы при сбое исходящей отправки |
| `TELEGRAM_IAP_SEND_TIMEOUT_MS` | 60000 | таймаут доставки одного входящего сообщения пиру |
| `TELEGRAM_NEW_TIMEOUT_MS` | 300000 | таймаут команды `/new` (5 мин) |
| `TELEGRAM_COMPACT_TIMEOUT_MS` | 300000 | таймаут команды `/compact` (5 мин) |

## Что задаёт iapeer

Эти приходят из окружения запуска, которое предоставляет iapeer; вручную вы их не задаёте.

| Переменная | По умолчанию | Значение |
|------------|--------------|----------|
| `IAPEER_ROOT` | `~/.iapeer` | корень состояния iapeer (хранилище ботов, манифест, логи) |
| `TELEGRAM_RUNTIME_IAPEER_BIN` | резолвится | путь к бинарю `iapeer` для команд управления |
| `IAP_BIN` | `iapeer` | бинарь для доставки сообщений пирам |

## Фиксированные значения

Эти зашиты и не настраиваются: сообщение разбивается на части по 4096 символов (32768 для rich-сообщений), вложение ограничено 50 МБ, голосовые файлы (`.ogg`/`.oga`) уходят как голосовые сообщения Telegram.
