# 03 — Боты и привязки

**Русский** · [English](../03-bots-and-bindings.md)

## Боты

Каждый агент общается через своего Telegram-бота. Боты регистрируются в мосте под коротким ключом и хранятся в каталоге рантайма (токен и настоящее имя бота — в его собственном файле настроек, плюс файл-замок, чтобы бота держал ровно один процесс).

```bash
iapeer connect telegram <агент> --token <токен>   # обычный путь: бот + привязка разом
```

Если нужно управлять ботами напрямую, у рантайма есть свои команды:

```bash
telegram-runtime bot add <ключ> --token <токен> [--username <запасное-имя>]   # зарегистрировать бота
telegram-runtime bot list [--json]                 # показать всех ботов
telegram-runtime bot remove <ключ>                 # удалить бота
```

При добавлении мост **проверяет токен у Telegram** и забирает оттуда настоящее имя бота (`@username`) — источник истины это Telegram, а не то, что ввёл оператор. Если сети нет, имя можно задать вручную запасным флагом. Удаление убирает бота целиком.

## Привязки

У моста нет отдельной базы связей — он выводит их из профилей пиров:

- **человек** в своём профиле несёт Telegram-идентификатор (`user_id`) — в какой аккаунт слать;
- **агент** несёт привязку к боту (`bot`) — через какого бота с ним общаются, и рядом — настоящее имя этого бота.

Прописывают эти поля две команды (их за вас вызывает `iapeer connect telegram`):

```bash
telegram-runtime interface human --user-id <id>          # задать Telegram-аккаунт человека
telegram-runtime interface bot <ключ> --peer <агент>     # привязать бота к агенту
```

Команды правят профиль аккуратно: дописывают только свои поля привязки, не трогая остальное содержимое профиля (имя, описание, природу, конфигурацию других плагинов).

## Несколько ботов под одним процессом

У разных агентов — разные боты, но опрашивает их всех **один процесс** моста. Каждого бота держит ровно один процесс (через файл-замок): если процесс перезапускается, замок переходит к новому, мёртвый замок снимается. Так несколько ботов работают параллельно, не мешая друг другу и не дублируясь.

## Проверка целостности

```bash
telegram-runtime doctor
```

Проверяет, что профиль на месте, Telegram-идентификатор человека задан, все боты настроены и привязаны к агентам, и нет ботов без агента. Это быстрый способ убедиться, что привязки согласованы.
