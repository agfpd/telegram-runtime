# 01 — Обзор

**Русский** · [English](../01-overview.md)

telegram-runtime даёт человеку присутствие в команде агентов через Telegram. Это рантайм присутствия для пира-человека — наравне с агентскими рантаймами Claude и Codex, только для человека, а не для AI. Владелец пишет агенту из мессенджера, агент отвечает туда же.

## Для агента человек — обычный пир

Ключевая идея: агент не работает с Telegram напрямую и не получает Telegram-инструментов. Для него человек — обычный пир с именем, которому он пишет одним и тем же `send_to_peer`:

```ts
send_to_peer(arthur, "…")
```

Мост принимает это сообщение и отправляет его через нужного Telegram-бота в нужный чат. Обратно — то же самое: сообщение человека в Telegram мост доставляет пиру-адресату как обычное входящее. Канал для агента прозрачен.

## Привязки выводятся из паспортов

У моста нет отдельной базы «кто с кем связан» — все привязки выводятся из профилей пиров (их паспортов):

- **человек** несёт в профиле свой Telegram-идентификатор (`interfaces.telegram.user_id`) — по нему мост знает, в какой аккаунт слать;
- **агент** несёт привязку к боту (`interfaces.telegram.bot`) — через какого бота с ним общаются.

То есть источник истины один — профили пиров; мост лишь читает их и строит маршрут. Подключение бота к агенту прописывает эти поля, отдельной синхронизации не требуется.

## Как ходит сообщение

**Человек → агент.** Человек пишет боту в Telegram. Мост принимает сообщение, понимает, от кого (по Telegram-аккаунту) и кому (по боту), при необходимости расшифровывает голос и разворачивает команды, и доставляет пиру-адресату.

**Агент → человек.** Агент вызывает `send_to_peer` на имя человека. Мост получает сообщение, находит Telegram-аккаунт получателя и бота отправителя, и шлёт сообщение в чат — текстом, а файлы и голос соответствующим типом.

## Что мост добавляет к простому обмену

Поверх доставки текста мост даёт живость общения:

- **голос** — расшифровывает голосовые от человека в текст, отправляет голосовые файлы от агента;
- **индикатор «печатает»** — показывает, что агент работает над ответом;
- **ленту активности** — по желанию владельца показывает, чем агент занят прямо сейчас;
- **команды управления** — позволяет из чата прервать, перезапустить или сжать сессию агента;
- **вложения** — переносит файлы и картинки в обе стороны.

## Связь с iapeer

telegram-runtime — это `*-runtime` пакет экосистемы [iapeer](https://github.com/agfpd/iapeer): рантайм присутствия, который iapeer ставит и онбордит как любой рантайм-пакет. Поэтому iapeer — обязательная зависимость: мост опирается на его обмен сообщениями (доставку пиру) и реестр (профили с привязками). Подробности — в [06 — Интеграция и установка](06-интеграция.md).
